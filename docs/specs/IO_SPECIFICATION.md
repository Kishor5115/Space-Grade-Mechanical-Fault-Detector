# I/O Specification — Space-Grade Mechanical Fault Detector

> **SSCS Chipathon 2026 — Track B (Sensor Circuits) | Team B22 — Team Space Jam**

---

## 1. Overview

The chip's external interface consists of:
- A **sensor SPI bus** (slave-side connection to the IIS3DWB MEMS accelerometer)
- A **DRDY interrupt** line from the sensor
- A **mode-select** pin (`tmr_forward_en`)
- A **single digital output** (`fault_flag_out`) — the fault alarm
- Standard **clock and reset** pins

The output is **not analog** and **not SPI** — it is a **single active-high digital level** that latches on first fault detection and remains asserted until explicitly cleared via the internal APB register write. There is no separate host-facing SPI port within the current `top.v` boundary.

## 2. Pin Table

| Pin Name | Direction | Width | Level | Description |
|---|---|---|---|---|
| `clk` | Input | 1-bit | 1.8V digital | System clock, 5–10 MHz. All flip-flops are positive-edge triggered. |
| `sys_rst_n` | Input | 1-bit | 1.8V digital | Active-low asynchronous reset. Asserts all FSMs to their idle/reset states simultaneously. |
| `c_miso` | Input | 1-bit | 1.8V digital | SPI MISO: IIS3DWB → ASIC. Raw digital from sensor; passed through a 2-stage FF synchronizer (`ff_2_sync`) before entering the SPI FSM. |
| `c_csn` | Output | 1-bit | 1.8V digital | SPI chip-select (active-low). Asserted by `spi_master` for each transaction. IIS3DWB requires CS to toggle between consecutive accesses. |
| `c_sclk` | Output | 1-bit | 1.8V digital | SPI bit clock (Mode 3: idles high). Generated by `clk_divider_5` (÷5 of system clock). Held high when `c_csn` is deasserted. |
| `c_mosi` | Output | 1-bit | 1.8V digital | SPI MOSI: ASIC → IIS3DWB. Drives boot write frames and read-command bytes. |
| `sensor_drdy` | Input | 1-bit | 1.8V digital | Data-Ready interrupt from IIS3DWB `INT1` pin (async, active-high level). Synchronized to core clock before use. Fires once per sensor sample at 26.667 kHz after boot. |
| `tmr_forward_en` | Input | 1-bit | 1.8V digital | Sample delivery mode select. **0** = Option A: samples accessible via local register poll only. **1** = Option B: additionally forward each sample into `tmr_reg_bank` over internal APB bus. Intended as a post-fab config fuse. Default (tie low) = Option A. |
| `fault_flag_out` | Output | 1-bit | 1.8V digital | **System alarm output.** Active-high, sticky. Asserts when any (frequency bin, sensor axis) magnitude exceeds the programmed threshold within a 512-sample block. Remains asserted until cleared by a `cfg_fault_clear` APB write (CTRL register bit 1). |

## 3. Sensor SPI Bus — Detailed Protocol

### SPI Mode
- **SPI Mode 3** (CPOL=1, CPHA=1): clock idles high; data driven on falling edge, sampled on rising edge
- Reference: IIS3DWB datasheet DS12569 Rev 8, Section 3.2 / Figure 3

### Clock Rate
- System clock: 10 MHz
- SPI clock: system clock ÷ 5 = **2 MHz** (within IIS3DWB's 10 MHz maximum)

### Transaction Types

#### Type 1 — Boot Configuration Write (at power-on only)

At reset release, `spi_master` executes the following 4 write transactions **before** looking for any `sensor_drdy`:

```
Frame format (16-bit, MSb first):
  Bit 15       : R/W = 0 (write)
  Bits 14–8    : 7-bit register address (MSb first)
  Bits 7–0     : 8-bit write data (MSb first)

Boot write sequence:
  1. CTRL1_XL    (addr=0x10, data=0xA0)  — Enable accelerometer at 26.667 kHz ODR, FS=00
  2. FIFO_CTRL4  (addr=0x0A, data=0x00)  — FIFO bypass (no on-sensor buffering)
  3. CTRL3_C     (addr=0x12, data=0x04)  — IF_INC=1 (enable auto-increment burst reads)
  4. INT1_CTRL   (addr=0x0D, data=0x01)  — Route DRDY to INT1 pin
```

After all four writes complete, `spi_master` enters `IDLE` and waits for `sensor_drdy`.

#### Type 2 — Burst Read (once per sensor sample, triggered by DRDY)

```
Transaction format:
  Phase 1 — Command byte (8-bit, CS asserted):
    Bit 7      : R/W = 1 (read)
    Bits 6–0   : Address = 0x28 (OUTX_L_A — starting register for XYZ burst)
    → full byte = 0xA8

  Phase 2 — Data receive (48-bit, auto-increment, CS still asserted):
    Bytes received in order: OUTX_L, OUTX_H, OUTY_L, OUTY_H, OUTZ_L, OUTZ_H
    → bit shift register fills MSb-first per byte, earliest byte at top:
    s_data_out[47:32] = {OUTX_H, OUTX_L}   (X-axis, 16-bit signed)
    s_data_out[31:16] = {OUTY_H, OUTY_L}   (Y-axis, 16-bit signed)
    s_data_out[15:0]  = {OUTZ_H, OUTZ_L}   (Z-axis, 16-bit signed)
```

### SPI Data Format (IIS3DWB Output)

Each axis output is a **16-bit two's complement signed integer** (Q1.15):
- Range: −32768 to +32767
- Value at ±FS (±2g): ±32768 (±1.0 normalized)
- Sensitivity: 1 LSB = FS/32768

The `axis_sequencer` sign-extends each 16-bit value into the 24-bit Q8.15 datapath used by `goertzel_core` without any explicit conversion module — the sign-extension is a combinational bit-replicate in the `goertzel_core`'s input registration always block:
```verilog
x_q15_r <= {{(DATA_W-SAMPLE_W){x_n[SAMPLE_W-1]}}, x_n};
```

## 4. Fault Output — Detailed Specification

### `fault_flag_out`

| Property | Value |
|---|---|
| Signal type | **Digital, active-high, sticky level** |
| Width | 1-bit |
| Voltage | 1.8V core digital |
| Default state | Low (0) — no fault |
| Set condition | Any of the 9 (axis×bin) magnitudes exceeds `cfg_threshold` within a 512-sample block |
| Clear condition | Software write of `1` to CTRL register bit 1 (`cfg_fault_clear`) via the internal APB bus |
| Persistence | Remains high across multiple blocks until explicitly cleared; a second fault while already flagged does NOT update `fault_mag_latched` / `fault_axis_latched` |

### Fault Status Registers

When `fault_flag_out` asserts, the following can be read from `tmr_reg_bank` via APB:

| Register | Address | Field | Description |
|---|---|---|---|
| STATUS | 0x14 | bit 0 | `fault_flag` — mirrors `fault_flag_out` |
| FAULT_MAG | 0x18 | [31:0] | `fault_mag_latched` — magnitude value that triggered the fault |
| FAULT_BIN | 0x1C | [1:0] | `fault_bin_latched` — frequency bin index that triggered (0/1/2) |
| FAULT_BIN | 0x1C | [3:2] | `fault_axis_latched` — physical axis that triggered (0=X, 1=Y, 2=Z) |

### Fault Attribution Example

If the X-axis exceeds threshold on frequency bin 0:
```
fault_flag_out   = 1
STATUS[0]        = 1
FAULT_MAG[31:0]  = <magnitude value>
FAULT_BIN[1:0]   = 2'b00  (bin 0)
FAULT_BIN[3:2]   = 2'b00  (X axis)
```

If the Z-axis exceeds threshold on frequency bin 2:
```
fault_flag_out   = 1
STATUS[0]        = 1
FAULT_MAG[31:0]  = <magnitude value>
FAULT_BIN[1:0]   = 2'b10  (bin 2)
FAULT_BIN[3:2]   = 2'b10  (Z axis)
```

## 5. Internal APB Register Map

The configuration and status interface is accessed via an internal APB bus (driven in silicon by a host-facing bridge outside `top.v`; driven directly in testbenches):

| Address | Register | Access | Description |
|---|---|---|---|
| 0x00 | CTRL | W | bit 0: `cfg_start` (W1P — write-1-to-pulse, enables `run_enable`); bit 1: `cfg_fault_clear` (W1P — clears sticky fault); bit 2: `cfg_stop` (W1P — disables `run_enable`) |
| 0x04 | CFG_C0 | R/W | [23:0] Goertzel coefficient for bin 0: `C0 = 2·cos(2π·f0/Fs)` in Q8.15 |
| 0x08 | CFG_C1 | R/W | [23:0] Goertzel coefficient for bin 1: `C1 = 2·cos(2π·f1/Fs)` in Q8.15 |
| 0x0C | CFG_C2 | R/W | [23:0] Goertzel coefficient for bin 2: `C2 = 2·cos(2π·f2/Fs)` in Q8.15 |
| 0x10 | CFG_THRESHOLD | R/W | [31:0] Fault detection threshold (unsigned integer, compared against `|X|²`) |
| 0x14 | STATUS | R | bit 0: `fault_flag` (sticky, cleared by CTRL[1]) |
| 0x18 | FAULT_MAG | R | [31:0] Magnitude that tripped the last fault (`fault_mag_latched`) |
| 0x1C | FAULT_BIN | R | [1:0] Frequency bin index (0–2); [3:2] Physical axis (0=X, 1=Y, 2=Z) |

### Coefficient Programming

The Goertzel coefficient for bin k tuned to frequency `f_k` (Hz) at sample rate `Fs = 26667 Hz` is:

```
C_k = 2 · cos(2π · f_k / Fs)

In Q8.15 fixed-point (24-bit signed, 1 sign bit, 8 integer bits, 15 fractional bits):
  C_k_q815 = round(C_k × 32768)

Example for common fault frequencies:
  f_k = 1 kHz  → C_k ≈ 1.9438  → C_k_q815 ≈ 63725  (0x0F8CD)
  f_k = 5 kHz  → C_k ≈ 0.7654  → C_k_q815 ≈ 25080  (0x061F8)
  f_k = 10 kHz → C_k ≈ -1.4142 → C_k_q815 ≈ -46341 (0xFFFF4ABB as signed 24-bit)
```

### Threshold Setting

`cfg_threshold` is compared against the computed magnitude `|X(f_k)|² = v1² + v2² - C·v1·v2`. The magnitude is a Q8.15 squared sum and grows with input amplitude. A typical threshold calibration:
1. Run with no fault stimulus → observe peak `FAULT_MAG` (noise floor magnitude)
2. Run with known fault amplitude → observe `FAULT_MAG` during fault
3. Set threshold between these two values

In the top-level testbench: threshold = 14, quiet floor ≈ 0, fault-tone magnitude >> 14.

## 6. Timing Characteristics

| Parameter | Value | Notes |
|---|---|---|
| System clock | 5–10 MHz | Lower end for margin in this process node |
| SPI bit clock | `clk / 5` | 1 MHz at 5 MHz clk; 2 MHz at 10 MHz clk |
| Sensor ODR | 26.667 kHz | Fixed by IIS3DWB boot configuration |
| SPI transaction duration | ~56 µs (boot write: 8 SPI clocks) / ~28 µs (burst read: 8 + 48 = 56 SPI clocks at 2 MHz) | Depends on SPI clock rate |
| Sensor DRDY period | 37.5 µs | = 1/26.667 kHz |
| DRDY-to-data-ready latency | ~28 µs | SPI burst read time |
| Block processing time | 19.2 ms | 512 samples × 37.5 µs/sample |
| Fault detection latency | ≤ 19.2 ms | Within one block after fault starts |
| `fault_flag_out` timing | Synchronous to `clk` | No additional synchronizer needed on output |
