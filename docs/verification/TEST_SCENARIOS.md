# Test Scenarios — Space-Grade Mechanical Fault Detector

> **SSCS Chipathon 2026 — Track B (Sensor Circuits) | Team B22 — Team Space Jam**

---

## 1. Overview

This document describes all test scenarios used to validate the Space-Grade Mechanical Fault Detector, organized by test suite. For each scenario, we specify:
- The SPI input stimulus (what the sensor model feeds into the ASIC)
- The internal configuration (APB register writes)
- The expected outputs (`fault_flag_out`, `FAULT_BIN` register contents)
- The verification method

All scenarios are automated self-checking tests using Icarus Verilog.

---

## 2. SPI Input Format Reference

Each scenario drives the IIS3DWB sensor model (`iis3dwb_model.v`) via the SPI bus. The model returns per-axis acceleration samples in the format:

```
SPI Burst Response (48-bit, MSb-first per byte, from sensor to ASIC):
  Byte 0: OUTX_L (X-axis low byte)
  Byte 1: OUTX_H (X-axis high byte)
  Byte 2: OUTY_L (Y-axis low byte)
  Byte 3: OUTY_H (Y-axis high byte)
  Byte 4: OUTZ_L (Z-axis low byte)
  Byte 5: OUTZ_H (Z-axis high byte)

ASIC shift register layout after reception:
  s_data_out[47:32] = {OUTX_H, OUTX_L}   ← X-axis (signed 16-bit Q1.15)
  s_data_out[31:16] = {OUTY_H, OUTY_L}   ← Y-axis (signed 16-bit Q1.15)
  s_data_out[15:0]  = {OUTZ_H, OUTZ_L}   ← Z-axis (signed 16-bit Q1.15)
```

Sample values are synthesized by the testbench using:
```verilog
// Q1.15 tone generation: x[n] = amplitude × sin(2π × f_bin0 × n / Fs)
sample = amplitude × sin(2π × f × n / Fs) × 32768
```
where `f_bin0 ≈ 1041.7 Hz` (block-coherent, 20 cycles per 512-sample block).

## 3. Scenario Suite 1 — SPI Master Protocol (`tb_spi_master_full.v`, 71/71 PASS)

### Scenario 1.1 — IIS3DWB Boot Configuration Writes

**Purpose:** Verify the sensor initialization sequence is correctly transmitted before any data reads.

| Register | Address | Write Data | Physical Meaning |
|---|---|---|---|
| CTRL1_XL | 0x10 | 0xA0 | Enable accelerometer, ODR = 26.667 kHz, FS = ±2g |
| FIFO_CTRL4 | 0x0A | 0x00 | FIFO bypass mode (no on-sensor buffering) |
| CTRL3_C | 0x12 | 0x04 | IF_INC=1: auto-increment address during burst reads |
| INT1_CTRL | 0x0D | 0x01 | Route DRDY signal to INT1 pin |

**SPI Input:** None (transmit-only frames; MISO is don't-care during write)

**Expected Output:** Four SPI write frames transmitted in order, each with the correct 16-bit frame pattern `{R/W=0, addr[6:0], data[7:0]}`. ASIC stays in boot state (`boot_active=1`) until all four writes complete.

**Verification:** `iis3dwb_model` captures each write event and the testbench asserts the exact address and data for all four writes (8 assertions of 71 total).

---

### Scenario 1.2 — DRDY-Triggered Burst Read

**Purpose:** Verify a complete 48-bit burst read from OUTX_L through OUTZ_H.

**SPI Input:**
- `sensor_drdy` asserted (active-high pulse) after boot completes
- `iis3dwb_model` returns pre-programmed 48-bit value `0x9B7619F701F6` on MISO

**Expected ASIC Behavior:**
1. `sensor_drdy` synchronized (2 clock cycles) → FSM exits IDLE
2. `c_csn` driven low
3. Command byte `0xA8` transmitted on `c_mosi` (MSb-first: R/W=1, addr=0x28)
4. 48 bits received on `c_miso`, MSb-first per byte
5. `s_data_out = 0x9B7619F701F6`
6. `s_data_out_valid` pulsed for 1 cycle
7. FSM holds in STOP until `core_ack` received, then returns to IDLE

**Expected Output:** `s_data_out[47:0] = 0x9B7619F701F6`

---

### Scenario 1.3 — Repeated Burst Reads (Back-to-Back DRDY Events)

**Purpose:** Verify the FSM correctly re-arms after each burst and processes a second DRDY.

**SPI Input:** Two DRDY pulses with different sensor data each time.

**Expected Output:** Two separate 48-bit values received correctly with correct handshaking.

## 4. Scenario Suite 2 — SPI-APB Interface (`tb_spi_apb_interface.v`, 8/8 PASS)

### Scenario 2.1 — Option A: Sample Local Register Access

**Configuration:** `tmr_forward_en = 0`

**SPI Input:** DRDY pulse → sensor returns `0x9B7619F701F6`

**APB Access Sequence:**
1. Read `STATUS` register (addr=0x0) → expect `{31'd0, 1'b1}` (data_ready=1)
2. Read `SAMPLE0` register (addr=0x4) → expect `32'hF701F6` → `s_data_out[31:0]`
3. Read `SAMPLE1` register (addr=0x8) → expect `{16'd0, 16'h9B76}` → upper 16 bits
4. Read `STATUS` register again → expect `{31'd0, 1'b0}` (data_ready cleared by SAMPLE1 read)

**Expected Output:** `fault_flag_out = 0` (detector not configured/started). APB bus remains idle throughout Option A operation (no `psel` assertion).

---

### Scenario 2.2 — Option B: TMR Forwarding via APB

**Configuration:** `tmr_forward_en = 1`, `tmr_sample_base = 32'h1000`

**SPI Input:** DRDY pulse → sensor returns `0x11912F898C62`

**Expected APB Transactions (automatically generated by `spi_apb_interface`):**
```
Write 1:  addr = 0x00001000 (tmr_sample_base + 0x0)
          data = 0x2F898C62  (s_data_out[31:0])

Write 2:  addr = 0x00001004 (tmr_sample_base + 0x4)
          data = 0x00001191  ({16'd0, s_data_out[47:32]})
```

**Verification:** `tmr_slave_stub.v` (APB slave stub) records all received write transactions. Testbench asserts exactly 2 writes occurred with the correct addresses and data.

## 5. Scenario Suite 3 — Goertzel Core Unit Tests (`tb_goertzel_core.v`, 7/7 PASS)

### Scenario 3.1 — Tri-Axis Independence Test

**Purpose:** Prove the three per-axis datapaths are independent (no crosstalk between X, Y, Z resonators).

**Configuration:**
```
bin 0 tuned to 1000 Hz:  C0 = 63725 (Q8.15)
bin 1 tuned to 5000 Hz:  C1 = 25080 (Q8.15)
bin 2 tuned to 10000 Hz: C2 = −46339 (Q8.15, off-target)
```

**SPI Input (synthesized, no real SPI):**
```
x[n] = 1.00 × (A1·sin(2π·1000·n/26667) + A2·sin(2π·5000·n/26667))
y[n] = 0.50 × (same two-tone signal)
z[n] = 0.25 × (same two-tone signal)
```
Applied for 500 samples (> 1 block; allows state to build up).

**Expected Output: Per-axis per-bin energy values after 500 samples**

| Axis | Scaling | Bin 0 (1 kHz, on-target) | Bin 1 (5 kHz, on-target) | Bin 2 (10 kHz, off-target) |
|---|---|---|---|---|
| X | 1.00× | ~828 (large) | ~5565 (large) | ≈0 (below threshold) |
| Y | 0.50× | ~174 (medium) | ~1391 (medium) | ≈0 |
| Z | 0.25× | ~970 (small) | ~348 (small) | ≈0 |

**Cross-axis ordering check:** X_energy > Y_energy > Z_energy for both on-target bins. A cross-wired axis would invert this ordering.

**Verification:** 7 assertions checking per-axis energy ordering, off-target bin near-zero, and `sample_done` count = 500.

## 6. Scenario Suite 4 — Full-Chip Integration (`tb_top.v`, 14/14 PASS)

### System Configuration (Applied Before All Cases)

```
APB Write: CFG_C0 = q815_coeff(1041.7 Hz) ≈ 63970  // block-coherent bin 0
APB Write: CFG_C1 = q815_coeff(5000.0 Hz) = 25080
APB Write: CFG_C2 = q815_coeff(10000.0 Hz) = -46341
APB Write: CFG_THRESHOLD = 14  // above quiet floor (~0); below fault-tone magnitude
APB Write: CTRL[0] = 1  // cfg_start → run_enable = 1
```

The threshold of 14 is calibrated by the testbench: quiet magnitude ≈ 0, fault-tone magnitude >> 14. This setting provides clear discrimination with no ambiguity.

---

### Scenario 4.1 — Case 1: Normal Operation (No Fault Stimulus)

**Purpose:** Confirm no false fault trigger on near-zero sensor data.

**SPI Input:**
```
model_outx = 0  (all samples)
model_outy = 0
model_outz = 0
```

**Expected Output:**
```
fault_flag_out = 0  (throughout 3-block observation window)
```

**Verification:** `check_true("Case1 Normal: no fault asserted (all axes quiet)", !fault_flag_out)` — PASS

---

### Scenario 4.2 — Case 2: Fault on X Axis Only

**Purpose:** Confirm correct fault detection AND correct axis attribution for X.

**SPI Input:**
```
model_outx = 0.8 × sin(2π × 1041.7 × n / 26667) × 32768  [Q1.15]
model_outy = 0   (silent)
model_outz = 0   (silent)
```

**Expected Output:**
```
fault_flag_out = 1  (within one block)
FAULT_BIN[1:0] = 2'b00  (bin 0 — the on-target bin)
FAULT_BIN[3:2] = 2'b00  (X axis = 0)
```

**Verification:**
```verilog
check_true("Case2 FaultX: fault_flag_out asserted", got_fault);      // PASS
check_eq2("Case2 FaultX: reported axis == X(0)", rd_val[3:2], 2'd0); // PASS
```

---

### Scenario 4.3 — Case 3: Fault on Y Axis Only

**Purpose:** Confirm correct fault detection AND correct axis attribution for Y. Also verifies that Y is not affected by the previous X-axis tone.

**Transition procedure:**
1. Set `amp_x = 0`, `amp_y = 0.8`
2. Wait 2 blocks (drains residual X-axis energy from previous block)
3. Clear any transitional fault from the drain period
4. Wait for fault from pure-Y stimulus

**SPI Input:**
```
model_outx = 0   (silent)
model_outy = 0.8 × sin(2π × 1041.7 × n / 26667) × 32768  [Q1.15]
model_outz = 0   (silent)
```

**Expected Output:**
```
fault_flag_out = 1  (after drain period)
FAULT_BIN[3:2] = 2'b01  (Y axis = 1)
```

**Verification:**
```verilog
check_true("Case3 FaultY: fault_flag_out asserted", got_fault);      // PASS
check_eq2("Case3 FaultY: reported axis == Y(1)", rd_val[3:2], 2'd1); // PASS
```

### Scenario 4.5 — Case 5: Simultaneous Multi-Axis Excitation (ITAG Showcase)

**Purpose:** Demonstrate the ITAG architectural advantage — detecting concurrent fault conditions on all three axes within a **single 512-sample block**, which the legacy axis-sequential design could not do.

**SPI Input:**
```
model_outx = 0.04 × sin(2π × 1041.7 × n / 26667) × 32768  [X, largest]
model_outy = 0.02 × sin(2π × 1041.7 × n / 26667) × 32768  [Y, medium]
model_outz = 0.01 × sin(2π × 1041.7 × n / 26667) × 32768  [Z, smallest]
```

All three axes simultaneously receive the same-frequency tone with decreasing amplitudes (4:2:1 ratio). All three amplitudes are chosen such that each individually exceeds the threshold (threshold = 14, all individual magnitudes >> 14).

**Expected Output:**
```
fault_flag_out = 1  (within one block from all-three-axes start)
FAULT_BIN[3:2] = 2'b00  (X axis = 0, first emitted and strongest)
cap_mag[0][0] != 0  (X axis bin 0 magnitude non-zero)
cap_mag[1][0] != 0  (Y axis bin 0 magnitude non-zero)
cap_mag[2][0] != 0  (Z axis bin 0 magnitude non-zero)
```

**Why this matters for spacecraft fault detection:**

In real spacecraft mechanical failure scenarios, a bearing failure on a reaction wheel may simultaneously excite vibration on all three structural axes. The ITAG design detects this within the current 19.2 ms block. The previous axis-sequential design would detect X-axis faults only on the X-designated block, Y-axis faults only on the Y-designated block, etc. — potentially taking up to 57.6 ms (3 full block rotations) to characterize the full fault signature.

**Verification:**
```verilog
check_true("Case5 Simul: fault_flag_out asserted (concurrent 3-axis)", got_fault);   // PASS
check_eq2("Case5 Simul: strongest/first axis X(0) attributed", rd_val[3:2], 2'd0);  // PASS
check_true("Case5 Simul: all 3 axes produced non-zero bin0 magnitude",
           (cap_mag[0][0] != 0) && (cap_mag[1][0] != 0) && (cap_mag[2][0] != 0));   // PASS
```

## 8. How to Reproduce

```bash
# From repository root, using Icarus Verilog
make sim_all

# Individual suites
make sim_spi        # 71/71
make sim_apb        #  8/8
make sim_goertzel   #  7/7
make sim_top        # 14/14
```

Each target produces a VCD waveform dump in the corresponding `testing/<block>/` directory for visual inspection with GTKWave or equivalent.
